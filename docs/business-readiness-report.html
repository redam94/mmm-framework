Executive summary

Marketing mix modeling is having a renaissance — privacy changes (ATT, cookie deprecation) gutted user-level attribution, and budgets are flowing back to aggregate measurement. But the category is flooded with black-box vendors selling correlational regressions dressed up as causal truth. CMOs have been burned: the model says "spend more on TV," they do, sales don't move, and nobody can explain why.

The MMM Framework is built around the one thing those tools quietly skip: causal rigor you can audit. Pre-registered analyses, experiments folded directly into the model's posterior, a fit-based refutation suite, and a structural-identification tool that proves an experiment will actually sharpen the answer before you spend a dollar on it. Wrapped around that engine is an adaptive measurement loop (fit → prioritize → experiment → recalibrate → reallocate → re-test) and a conversational analyst agent that operates the whole workflow — collapsing the "you need a Bayesian statistician on staff" barrier that keeps rigorous MMM out of the mid-market.

Readiness in one line

The hardest, riskiest part of becoming a multi-tenant SaaS — real authentication, tenant isolation, and a buyer-facing security posture — is Shipped. All four P0 table-stakes are shipped (auth/tenancy, trust collateral, security hardening, and pricing/packaging); what remains is billing integration and the go-to-market proof a first design partner provides. In other words: the science and the security are de-risked; the work ahead is business, not research.

The opportunity

Why the category is moving

Aggregate marketing measurement is structurally back in demand: the privacy stack that powered a decade of click-attribution is gone, regulators keep tightening, and finance wants a defensible number for the largest discretionary line on the P&L. MMM is the surviving primitive.

Why incumbents leave a gap

  • Legacy panels & agencies (e.g. Nielsen-style) are black boxes — slow, expensive, and impossible for a client to interrogate.
  • Open-source toolkits (e.g. Robyn, Meridian) are capable libraries with no product layer, no workflow, and no causal-defensibility scaffolding — they assume a data-science team.
  • The honest middle is empty: an auditable, causal measurement system — not a chart, not a library — that a mid-market or agency analyst can actually operate.

That empty middle is the wedge: rigor + workflow + an agent that makes it usable.

The product

One platform runs the entire marketing-measurement loop — and an agent does the work as you chat.

1 · Causal Bayesian engine

A production Bayesian MMM extending PyMC-Marketing: per-channel adstock (geometric, delayed, Weibull) and saturation (logistic, Hill, Michaelis-Menten, tanh), hierarchical geo & geo×product panels, and extended models for funnel mediation, multi-outcome, and combined effects. Fast NUTS via NumPyro/NutPie.

2 · The measurement loop (T₀–T₅)

Fit → rank experiments by expected information gain in dollars (EIG/EVOI) → run pre-registered, lifecycle-tracked experiments → fold readouts back into a calibrated refit → reallocate budget → re-test as evidence decays. Measurement becomes a flywheel, not an annual report.

3 · The analyst agent

A LangGraph agent with ~40 tools validates data, runs EDA, configures and fits models, reads out ROI/decomposition/saturation, runs the budget optimizer, and plans/calibrates experiments — executing Python in a sandboxed kernel and producing branded client reports.

Capability surface

Experiment economics & design

  • Model-anchored incremental ROAS with a powered / underpowered verdict
  • Opportunity cost & spend-at-risk of a holdout (a holdout can be net-positive)
  • A/A·A/B simulation that corrects for autocorrelation-inflated false positives
  • A Pareto-front optimizer trading MDE × power × cost × duration, with cool-down windows
  • Geo-lift, matched-market DiD, and budget-neutral national flighting designs

Platform & delivery

  • Modern React UI (Program / Experiments / Performance / Agent / Knowledge / Team)
  • FastAPI + Redis + ARQ backend that queues long model fits asynchronously
  • Per-project RAG knowledge base — answers quote the actual client brief
  • One-click branded HTML client reports in the client's colors
  • Bring-your-own LLM (Anthropic, OpenAI, Gemini, Vertex, or local LM Studio); open-source Python library underneath

Platform status is Beta (web UI, agent workspace, and planning engine on the Beta tier). The React UI is the supported surface; a legacy Streamlit UI exists as a secondary surface.

Evidence & validation

Credibility is the product. Here is what is measured — labeled honestly, because overclaiming is how MMM vendors lose technical buyers.

Proof pointResultBasis
Out-of-time forecast accuracy 3.0% MAPE (MASE 0.24) — ~¼ the error of a "copy last year" baseline (13.8%). 80%/95% interval coverage at 90.4% / 94.2%. Synthetic benchmark Genuine out-of-time test (expanding-window refit + posterior forward-replay) on our 156-week, 7-channel benchmark world.
Real external data (Lydia Pinkham, 54 yrs) Carryover sided with the short-duration literature; total-effect ROAS 1.48 [1.12, 1.85] landed inside the bracket six decades of published advertising econometrics produced. Real data A causal-anchor win on a genuinely external, decades-old dataset.
Honesty under stress On that same real series it reported no forecast skill over naïve persistence (MASE 1.36) while keeping intervals calibrated (94% coverage). A regime-shift world degrades the benchmark to 11.4% MAPE on the identical protocol. Real + designed-to-fail "A backtest that cannot fail cannot validate." The system tells you when it doesn't know.
Speed NumPyro NUTS measured 6.0× faster than stock PyMC at equal draws; a national fit completes in ~17 s. Measured Comparable convergence (R-hat 1.010 vs 1.008).
Methodological rigor Refutation suite, in-graph experiment-calibration likelihoods, and a Fisher/Laplace structural-identification design that proves an experiment will move the saturation/adstock/coefficient parameters off their priors. Shipped capability Addresses the equifinality ridge that makes these parameters weakly identified from smooth spend.

Reading these numbers honestly (so the pitch survives diligence)

  • The headline 3.0% MAPE is from our synthetic benchmark world, not a client engagement — present it as such.
  • The only genuinely-external dataset validated is Lydia Pinkham. It is a strong causal win and an honest forecasting non-win — do not claim we beat naïve forecasting on real data; we explicitly did not.
  • Forecast accuracy validates the predictive model, not attribution correctness. A model can forecast well and attribute wrongly — which is exactly why the causal scaffolding matters.

P0 — Commercial table stakes

The minimum to responsibly sell to a single company. Two of four are done; the security mechanisms below were built and adversarially verified, not asserted.

P0.1 · Authentication, multi-tenancy & RBAC Shipped

  • Org-scoped identity: an Organization tenant layer above projects; scrypt password hashing (constant-time verify); stateless HS256 JWT access/refresh tokens; a pluggable verifier so an external OIDC/SSO IdP plugs in later without rewriting call sites.
  • Tenant isolation enforced on both backends: require_project_access on all 22 project routes plus a session-guard factory that closed ~35 additional thread-keyed holes an internal audit surfaced; cross-tenant access returns 404 (not 403) so existence can't be probed. IDOR-resistant by-id routes; org-scoped list endpoints; a dedicated cross-tenant test matrix.
  • Full account lifecycle: admin invites, self-service password reset (no email enumeration), admin deactivation, and refresh-token single-use rotation + revocation — all emitting structured audit events tagged with user + org.
  • Frontend wired: JWT store, login, bearer interceptor, single-flight refresh-or-logout; typecheck and build clean.

P0.2 · Security & trust collateral Shipped

Procurement is de-risked before the first call: a buyer-facing Trust & Security page, a SOC 2 readiness map, a SIG-lite security questionnaire, a data-processing & sub-processors summary, and an RFC 9116 security.txt + responsible-disclosure page. Every control cites source, and an adversarial verification pass caught and corrected a real overclaim — the credibility posture a security reviewer rewards.

P0.3 · Security-hardening tail Shipped

  • Instant access-token kill: a token_version claim checked per request — deactivation, password reset, and "sign out everywhere" reject every live access and refresh token immediately, not just on next refresh.
  • Rate limiting: a per-org limiter on the expensive routes (chat, model fits, KB ingest) and an IP-based brute-force throttle on the unauthenticated login/signup/refresh endpoints. Off by default; verified by tests.
  • Sandbox image shippable: one-command build/verify/push (make kernel-image) + a runbook tying it to MMM_AGENT_HOSTED=1; the Containerfile and k8s manifests (gVisor, egress-deny) already existed. Build/push is a deploy-env step.
  • Hardened against an adversarial review that found (and we fixed) the brute-force gap and missed expensive routes.

P0.4 · Pricing & packaging Shipped

Open-core model: a free, self-hosted OSS core; paid managed tiers whose differentiators are exactly the multi-tenancy / SSO / hosted-sandbox capabilities already built — so the security investment is the monetization.

  • Enforceable entitlements, not just a doc: four tiers (Free / Team / Business / Enterprise) with seat/project/monthly-fit limits and a feature-gating map; per-org usage metering (GET /auth/usage); limits enforced (seats at invite-accept, projects at create, premium features → 402). Tested.
  • Collateral: a pricing page (tiers match the code exactly), a per-fit cost model grounded in the measured ~17 s national fit (≈$0.011 COGS/fit → 80%-margin price floors), and a packaging one-pager.
  • Remaining (Track 3): billing integration (Stripe/webhook → plan changes) and hard fit-quota enforcement — today fits are metered, not blocked.

P1 — Make it stick

Once you can sell it, these convert a sale into retention and references.

Self-serve onboarding Shipped

A 6-step path-to-first-model checklist — an API computed from real project state (brief → data → fit → review → experiment) and a guided front-end flow on the home page (progress bar, next-step hint, workspace CTA) that auto-hides once complete. Once data is added it surfaces the agent's EDA data-quality verdict inline (ready-to-fit / warnings / blocking errors), so a new user fixes problems before fitting — without an analyst.

Reliability & observability Shipped

A tamper-evident, hash-chained audit log with structured, queryable events, a Business-tier audit export, and an off-host shipper (background tick → a configurable remote sink, so the log isn't the only copy — closing a SOC 2 gap). A GET /observability endpoint surfaces audit-chain integrity, ship backlog, and recent fit activity. A customer-facing fit-SLA dashboard is the remaining UI follow-on.

Reference customer + ROI case Partial

Shipped (enablers): a founder-sendable design-partner program and an honest ROI case-study template (placeholders, not fabricated numbers) mapped to the T₀–T₅ loop. Needed: a signed design partner to turn the Pinkham/backtest credibility into a named ROI story — a sales motion, not code.

Compounding levers once the motion exists.

Integrations Partial

Portfolio benchmarking & governance Shipped

Shipped: an org-scoped cross-brand benchmark API (GET /portfolio-benchmark) over the per-run history — ranks each brand's channel ROIs against the portfolio (median / p25–p75 / percentile), with governance signals (model freshness/staleness, calibration coverage) — plus a full Constellation dashboard (React) rendering it: governance tiles, a per-channel ROI box-plot with every brand scattered as a dot, and a brand table with fit recency, portfolio mROI, and vs.-portfolio leader/laggard counts. The agency/holding-co land-and-expand view, end to end.

"Model-defense" report Shipped

End-to-end: a one-click agent tool runs the causal refutation suite on the fitted model, reads convergence and any experiment calibration, and emits a CFO-readable verdict (Robust / Qualified / Needs scrutiny) with per-test plain-English explanations and honest caveats — a branded, standalone HTML page (served at /model-defense). Turns the platform's deepest differentiator, causal rigor, into a sales artifact.

Business model & go-to-market

How it monetizes

  • Seats for analysts + per-project / per-fit compute (compute is the real COGS — meter it).
  • A services / onboarding tier: the synthetic-world tooling and branded-report generator make white-glove cheap to deliver.
  • Land-and-expand: start with one brand, expand across a portfolio — the geo × product panel support is built for exactly this.

Who buys, and why now

  • Mid-market brands and agencies priced out of legacy panels and underserved by open-source toolkits.
  • The wedge is the agent: it lets a generalist analyst operate rigorous MMM — expanding the market below the enterprise data-science tier.
  • Privacy-driven demand for defensible aggregate measurement is structural, not a fad.

Risks & honest limitations

Stated plainly — because a readiness report that hides these isn't credible, and because every item is tracked.

  • Default posture is single-tenant. Authentication is off by default; the out-of-box install runs a dev principal that bypasses scoping. Multi-tenant isolation activates with MMM_AUTH_ENABLED=1 — by design, but it means the secure posture is opt-in.
  • The code sandbox is deployment-dependent. The hardened container kernel is inert until its image and a runtime are deployed; the in-process and subprocess kernels are not sandboxes. (A half-applied sandbox was deliberately made to fail closed.)
  • Some platform plumbing is unfinished: the S3 storage backend is a settings placeholder (local filesystem is implemented); retention purges only job records, not on-disk data; encryption in-transit/at-rest is deployer-supplied (TLS proxy + encrypted volumes), not a platform feature.
  • No SOC 2 certification — the collateral is explicitly readiness, not certified; no audit has occurred.
  • Validation is mostly synthetic. The headline accuracy numbers are benchmark-world; the one real external dataset is a causal win but a forecasting non-win. A design partner is the path to a real-client ROI claim.
  • Tenant isolation is application-code-enforced over SQLite + JSON sidecars, not datastore row-level ACLs — appropriate at this stage, worth migrating as scale grows.

The ask — next 90 days

What closes the gap to "sellable"

  1. Finish the P0 tail: per-org quotas, instant token-kill, and ship the sandbox image (≈2–3 weeks).
  2. Package & price: tiers, per-fit cost model, and metering — the only P0 item still untouched (≈3–4 weeks).
  3. Land one design partner: a single named brand/agency to convert the synthetic + Pinkham credibility into a real ROI case study — the highest-leverage asset for the next raise and the first ten sales.

The science is validated and the security is built. The remaining work is commercial packaging and a first reference — both bounded, both underway or scoped.
