Experiment Playbook

An MMM tells you where the model is uncertain; an experiment is how you buy the missing information. This page is the practical playbook: every experiment type the framework supports, how to plan each one with honest power math, how its readout flows back into the MMM as a calibration likelihood, and how the value-of-information machinery — EIG, EVOI, and the net dollar value of testing — decides which experiment (geo or not) is actually worth running.

Why experiments and an MMM need each other

Observational media data is confounded by design: budgets chase demand, seasonality moves spend and sales together, and channels launch alongside promotions. The MMM controls for what it can see, but some channel effects are only weakly identified from history alone — the posterior stays wide, and a wide posterior on a big budget line is an expensive place to be undecided.

Experiments inject the identifying variation the history lacks. The framework treats them as a first-class loop, not a side quest:

  1. Fit the MMM and find the decision-critical uncertainty (EIG/EVOI grid).
  2. Design the test with the method that fits your data (method registry), powered against the model's own predicted effect.
  3. Price it: short-term opportunity cost vs the reallocation gain the learning unlocks (net value).
  4. Run & read out with a pre-registered analysis plan.
  5. Calibrate: the readout becomes a likelihood term in the next fit (calibration), shrinking the posterior where the test actually measured.
  6. Re-test when the information decays (channel half-lives; the grid flags retest due).

The methodology of the loop itself — priors, likelihood routes, decay — is covered in The Calibration Loop. This page is about the experiments: which one, how to size it, and how to get its value back into the model.

Choosing a method — the experiment-method registry

Every supported methodology is a named entry in the method registry (planning/methods/), carrying its data requirements, estimator, power math and references. The design tools, the REST API and the agent all enumerate the same registry, so "which methods can my data support?" has one answer everywhere: 

from mmm_framework.planning.methods import list_methods, methods_for_data

for spec in list_methods():
    print(spec.key, "—", spec.name, "| family:", spec.requirement.family)

# only the methods THIS dataset can support (geo count, pre-period length…)
usable = methods_for_data(n_geos=8, n_pre_weeks=52, has_panel=True)

Matched-pair geo lift Yes (pairs) ≥ 4 geos, geo panel Incremental KPI / iROAS in treated geos You can randomize; the default gold standard for geo data. Holdout (go dark) or scaling (spend change) cells.
Synthetic control (SCM) No (observational) ≥ 4 geos, ≥ 12 pre-weeks Treated-market lift vs a convex donor counterfactual One or few treated markets you could not randomize; inference by RMSPE-standardized placebo permutation.

TBR / CausalImpact (BSTS) Either ≥ 2 geos, ≥ 12 pre-weeks Cumulative lift with a full Bayesian posterior Few geos, strong control series; the fast conjugate tier powers simulations, the full structural time-series the headline readout.
Geo-based regression (GBR) Yes ≥ 4 geos, ≥ 8 pre-weeks Treatment coefficient in a post-on-pre cross-section Many geos, short windows; one row per geo keeps it robust to week-level noise.

One dispatcher. design_experiment(..., method="synthetic_control") infers the right design family from the method, attaches the method's metadata to the plan, and rewrites the analysis-plan estimator line — so the pre-registered plan and the eventual readout use the same estimator by construction.

Planning a geo experiment

Geo designs start from design_options (what does this dataset support?) and design_experiment (build the runnable plan): matched pairs by pre-period KPI similarity, randomized treatment within pair, a power curve of minimum detectable effect (MDE) by duration, and DiD power + placebo diagnostics. 

from mmm_framework.planning import design_experiment, design_options

opts = design_options("data.csv", kpi="Sales", channel="TV")
print(opts["designs"], opts["methods"])   # families + registry methods this data supports

plan = design_experiment(
    "data.csv", "Sales", "TV",
    method="synthetic_control",   # or tbr / gbr / did_mmt / regadj_geo
    design="holdout",             # or "scaling" with a signed intensity_pct
    duration=8,
)
print(plan["mde_roas"], plan["power_curve"])

When a fitted model is available, the plan is anchored: the model predicts the incremental ROAS the test should see (perturbing only the treated geos over the test window, not the global curve) and issues a powered / underpowered / overpowered / inconclusive verdict, plus the opportunity cost of deviating from business-as-usual — including the case where a holdout saves money. A cool-down period derived from the channel's fitted adstock tells you how long treated geos need before they are back to baseline.

Ghost ads — user-level power with no panel at all

Ghost ads randomize individuals: treated users see the real ad; ghost/PSA users a placebo the ad server logs as "would have been served". Because the randomization lives in the ad server, this method needs no geo panel and works pre-fit — it is a standalone power calculator over reach, baseline rate and exposure: 

from mmm_framework.planning.methods import (
    GhostAdsDesign, ghost_ads_power, ghost_ads_simulate, ghost_ads_users_for_mde,
)

design = GhostAdsDesign(
    users_reached=400_000,
    baseline_rate=0.021,          # control conversion rate
    treated_fraction=0.5,
    exposure_rate=0.65,           # share of assigned users actually reached (ITT dilution)
    value_per_conversion=38.0,    # enables the break-even lift + ROAS bridge
    cost_per_user=0.05,
)
power = ghost_ads_power(design)          # closed-form MDE + diagnostics
# users required to detect a +0.2pp absolute lift (≈ +10% relative on 2.1%)
need = ghost_ads_users_for_mde(design, target_lift_abs=0.002)
check = ghost_ads_simulate(design)       # binomial Monte-Carlo: empirical FPR + power
  • ITT vs TOT. With 65% exposure, the intent-to-treat effect is diluted by the never-reached — the calculator reports both scales so you don't power the test against the wrong one.
  • Rare events. Below ~30 expected conversions per arm the normal approximation is flagged as shaky; the simulation check is the honest fallback.
  • Economics. With a value per conversion (or a CLV-derived lifetime value) the calculator reports the break-even lift — the smallest true effect at which the media pays for itself.
  • Cost per conversion — never a naive flip. Clients read results as CPA (spend ÷ incremental conversions), but the reciprocal of a well-behaved lift estimate is extremely right-skewed: its mean overshoots, the symmetric delta-method interval under-covers (and can dip below zero), and a weak readout's honest interval is unbounded above. The planning summary that survives inversion is the maximum detectable cost per conversion — cost ÷ the detectable-lift bound — reported by ghost_ads_power directly, with planning.cpa (cpa_interval, cpa_power, simulate_cpa_distribution) supplying the honest interval, target-certification power, and the skew demonstration for any lift-based design.

Switchbacks — time-randomized on/off blocks

When there is no geo split and no user-level randomization, you can still randomize time: switch the channel on and off in randomized blocks. Two things make naive switchback power calculations dishonest, and the design handles both: 

from mmm_framework.planning.methods import switchback_design, switchback_power

design = switchback_design(
    "data.csv", "Sales", "Search",
    duration=12,
    amplitude_pct=50.0,
    # block length defaults to the channel's carryover; pass mmm=... to use
    # the FITTED adstock washout instead of the heuristic
)
power = switchback_power(design, sigma_y=noise_sd)
print(power["se_iid"], power["se_honest"])   # the honest SE carries the AR(1) design effect
  • Carryover. A block shorter than the channel's adstock memory smears treatment into control blocks. The block length is derived from the fitted washout (cooldown_weeks) when a model is available, and the design warns when the schedule cannot respect it.
  • Autocorrelation. KPI series are serially correlated, so the effective sample size is smaller than the block count. The design effect uses the AR(1) coefficient of the detrended levels — first-differencing a smooth series flips the estimated correlation negative and silently zeroes the correction.

National flighting — the design that identifies the curve

Budget-neutral flighting schedules the channel through planned spend levels. A two-level on/off pins the average ROAS; a schedule with three or more distinct levels traces the response curve itself, so the marginal ROAS and saturation become identifiable — the quantities the budget optimizer actually uses. The optimizer reports power separately for ROAS, contribution and marginal ROAS, and a two-level design is never credited with curve power its secant cannot deliver. See the design optimizer for how the level grid is chosen.

The methodology leaderboard — audition estimators on your own history

Analytic power formulas assume away the mess in real panels: autocorrelation inflates false-positive rates, and a method that looks powered on paper can be invalid on your data. The leaderboard runs every estimator the data supports through A/A (null) and A/B (injected-lift) simulations on your own history, then ranks by validity → power → cost: 

from mmm_framework.planning import methodology_leaderboard

board = methodology_leaderboard(
    "data.csv", "Sales", "TV",
    mmm=None,          # pre-fit works; pass the fitted model to inject ITS predicted lift
    duration=8,
)
# each row: method, empirical false-positive rate (A/A), empirical power (A/B),
# calibrated critical value, and a recommendation

The A/A pass is the part most teams skip and shouldn't: an estimator whose 5% test rejects 15% of null windows will "detect" lift that isn't there, and only a block-calibrated critical value restores honest size.

Calibrating the MMM with the readout

A finished experiment is worth more than its own read — it is direct evidence about a channel's causal effect that the next MMM fit should not be allowed to ignore. The framework folds readouts into the model as likelihood terms on the model's own estimands: the measured value, with its standard error, constrains the in-graph contribution / ROAS / marginal-ROAS of the tested channel over the tested window. 

from mmm_framework.calibration import ExperimentMeasurement, ExperimentEstimand

exp = ExperimentMeasurement(
    channel="TV",
    test_period=("2025-03-03", "2025-04-28"),
    value=1.8,                        # measured incremental ROAS
    se=0.35,                          # the experiment's uncertainty — never drop it
    estimand=ExperimentEstimand.ROAS,
)
mmm.add_experiment_calibration([exp])
results = mmm.fit()                   # the posterior now respects the experiment

Why this route beats hand-tightening a prior:

  • The estimand matches. A geo test measures incremental KPI over a window — that is a statement about the model's contribution in that window, not about a raw coefficient. The likelihood is placed on exactly the quantity the experiment measured (CONTRIBUTION, ROAS, or MROAS with the spend-lift percentage for scaling cells).
  • Uncertainty propagates. A noisy test moves the posterior a little; a precise one moves it a lot. No cliff-edge overrides.
  • Disagreement is visible. If the model and the experiment conflict, the tension shows up in fit diagnostics instead of being silently overwritten — that disagreement is information about model misspecification.

Every method in the registry calibrates the same way, because every readout reduces to (estimand, value, standard error, window):

  • Geo methods (matched-pair, SCM, TBR, GBR, DiD-MMT) read out incremental KPI or iROAS directly; holdout_regions restricts the likelihood to the treated geos on a geo model.
  • Ghost ads: per-user lift × users reached × value per conversion is a window contribution (or divide by test spend for ROAS); the delta-method SE comes with it. A user-level RCT calibrates a national MMM exactly like a geo test does.
  • Switchbacks and flighting read out an on/off or level contrast — an ROAS (two levels) or a marginal ROAS (≥ 3 levels, MROAS with spend_lift_pct).

In the Oracle workflow the same flow is conversational: record_experiment_readout stores the result against the pre-registered experiment, apply_experiment_calibration stages it into the model spec, and the next fit_mmm_model marks the experiment calibrated in the lifecycle registry.

Off-panel calibration — tests from outside the training window

An experiment that ran before your modeling window (or in geos the panel doesn't carry) still calibrates. Supplying the spend level the test ran at routes the likelihood through the channel's global response curve — evaluating β·sat(adstock(s)) at that spend — instead of summing training-matrix rows, so window overlap becomes irrelevant: 

exp = ExperimentMeasurement(
    channel="TV",
    test_period=("2023-05-01", "2023-06-26"),  # predates the training window
    value=210_000.0, se=60_000.0,
    estimand=ExperimentEstimand.CONTRIBUTION,
    eval_spend=45_000.0,     # per period, per treated unit, during the test
    eval_periods=8,
    eval_units=12,           # treated geos
    adstock_state="steady_state",   # or "cold_start" for a from-zero launch
)

The assumption you are buying is structural stationarity — the response curve is stable between the test period and the training period. State it, don't hide it; the further back the test, the more the information-decay clock should discount it anyway.

EIG & EVOI — which uncertainty is worth paying to remove

Not every wide posterior deserves an experiment. Two numbers separate the cases:

EIG (expected information gain) — how much a test of a given precision would teach. For a channel with ROI posterior sd \( \sigma_k \) and an experiment that measures ROI with sd \( \sigma_{\text{exp}} \):

$$ \mathrm{EIG} = \tfrac{1}{2}\,\ln\!\left(1 + \sigma_k^2 / \sigma_{\text{exp}}^2\right) \ \text{nats} $$

EVOI (expected value of information) — what that learning is worth to the budget decision: simulate outcomes the experiment could return, reweight the posterior by each, re-optimize the allocation, and score the expected improvement over deciding without the test (preposterior analysis). EVOI is capped by EVPI, the value of perfect information. A channel can have huge EIG and near-zero EVOI — you'd learn a lot, but the budget wouldn't move. 

from mmm_framework.planning import compute_experiment_priorities

grid, portfolio = compute_experiment_priorities(mmm)
for row in grid:
    print(row.channel, row.quadrant, row.eig, row.evoi, row.retest_due)
# quadrants: test_now / learn_cheaply / monitor / deprioritize

The critical honesty rule: \( \sigma_{\text{exp}} \) must be design-grounded — derived from the experiment you could actually run (geo footprint, reach, blocks), never from the channel's own posterior width, or every channel scores the same and the ranking collapses. Evidence also decays: each channel class has an information half-life, and once the effective uncertainty regrows enough that a fresh test would clear the EIG threshold, the grid flags retest due.

Selecting non-geo experiments with the same machinery

Nothing in EIG or EVOI is geographic. Both consume exactly one fact about a candidate design: the standard error \( \sigma_{\text{exp}} \) it would achieve on the ROI scale. Every planner on this page emits one:

DesignWhere its \( \sigma_{\text{exp}} \) comes from
Geo holdout / scaling Power curve MDE at the chosen duration: \( \sigma_{\text{exp}} = \mathrm{MDE}/2.8 \) (the 80%-power, 5%-size factor \( z_{0.975}+z_{0.80} \)).
Ghost ads The two-proportion SE of the lift, bridged to ROAS by value per conversion and test spend — ghost_ads_power reports the MDE; divide by 2.8.
Switchback switchback_power's se_honest (AR(1)-corrected), already an effect-scale SE.
National flighting The design-matrix estimand SEs — se_roas for the average, se_mroas when ≥ 3 levels identify the curve.
No design yet (screening) sigma_exp_for_design("national_pulse", roi_median) — operational relative-precision defaults per design class. 
from mmm_framework.planning import compute_experiment_priorities
from mmm_framework.planning.eig import sigma_exp_for_design

# a ghost-ads test is on the table for Search; a switchback for Display:
grid, portfolio = compute_experiment_priorities(
    mmm,
    design_type="national_pulse",            # default for channels with no override
    sigma_exp_overrides={
        "Search": ghost_ads_mde_roas / 2.8,   # from ghost_ads_power + the value bridge
        "Display": switchback_se_roas,        # switchback_power()["se_honest"], ROI scale
    },
)

The grid then compares a user-level RCT on Search against a geo holdout on TV against a switchback on Display on one scale: what each would teach (EIG) and what that teaching is worth to the allocation (EVOI). This is the point of routing everything through \( \sigma_{\text{exp}} \) — the experiment-selection problem stops caring what kind of experiment it is.

Net test value — is this experiment worth the money?

EVOI prices the upside. A real test also has a downside: the profit given up while it runs (a holdout forgoes sales; a scaling test commits budget). The net-economics module produces the single decision figure:

$$ \text{net value} \;=\; \underbrace{\min(\mathrm{EVOI},\ \mathrm{EVPI}) \cdot m \cdot \bar d}_{\text{decayed reallocation gain (\$)}} \;+\; \underbrace{\mathbb{E}[\,m\,\Delta\mathrm{KPI} - \Delta\mathrm{spend}\,]}_{\text{signed profit during the test}} $$

where \( m \) is the margin per KPI unit and \( \bar d = \tfrac{1}{H}\sum_{w < H} 0.5^{\,w/h}\,\rho^{w} \) is the average informativeness retained over the decision horizon \( H \) under the channel's information half-life \( h \) (and optional financial discounting \( \rho \)) — EVOI values a posterior that stays sharp forever; real information doesn't. 

from mmm_framework.planning import compute_experiment_net_value

nv = compute_experiment_net_value(
    channel="TV",
    evoi_kpi_units=evoi,             # from the priorities grid / anchor loopback
    evpi_kpi_units=portfolio["evpi"],
    margin_per_kpi=0.42,
    response_horizon_weeks=26,
    half_life_weeks=52.0,            # TV evidence decays slowly
    opportunity_cost_result=oc,      # reuses its per-draw KPI deltas → P(net > 0)
)
print(nv.net_value, nv.prob_net_positive, nv.breakeven_horizon_weeks)

The signed convention keeps a money-saving holdout coherent: when the saved spend outweighs the forgone margin, the "loss" is negative, the net value exceeds the gain, and the break-even horizon is zero — the test pays for itself on day one.

The design optimizer — a Pareto front priced in dollars

Within one channel there is still a design space: holdout vs scaling, footprint, intensity, duration, spend levels. suggest_experiment evaluates a bounded grid of candidate designs on four objectives — MDE, power shortfall below the 80% target, cost, and duration — and returns the non-dominated Pareto front plus a recommended "knee" design with runnable test/control groups (or a flighting schedule) and the adstock-derived cool-down.

When a margin is known, the cost axis is upgraded from "what does the test destroy short-term" to the net value of testing — gain minus loss, per candidate. The expensive part of that is EVOI, which normally costs a preposterior Monte-Carlo per design. The optimizer instead prices the whole grid from two anchored MC evaluations using a calibrated Gaussian surrogate (Raiffa–Schlaifer preposterior form):

$$ \mathrm{EVOI}(\sigma) \;\approx\; k \cdot s(\sigma)\,\Psi\!\big(\delta / s(\sigma)\big), \qquad s(\sigma) = \tau\sqrt{\tau^2/(\tau^2+\sigma^2)} $$

where \( \tau \) is the channel's prior ROI sd, \( s(\sigma) \) the sd of the preposterior mean (how far a test of precision \( \sigma \) can move the decision), \( \Psi \) the unit-normal loss integral, and \( (k, \delta) \) are fitted to the two MC anchors placed at the extremes of the grid's design precisions — so every candidate interpolates inside the calibrated bracket, where the surrogate tracks the exact preposterior MC to within ~15% even for distinctly non-Gaussian ROI posteriors. 

from mmm_framework.planning import suggest_experiment

out = suggest_experiment(
    mmm, "data.csv", "Sales", "TV",
    margin=0.42,                 # enables the net-value axis
    duration_min=6, duration_max=16,
    intensity_min=-100,          # go-dark holdout …
    intensity_max=100,           # … through +100% scaling
)
best = out["recommended"]
print(best["net_value"], best["evoi_kpi"], best["treatment_geos"], out["cooldown"])

The front then reads the way a CFO wants it read: designs that are precise, powered, fast, and expected to create money net of their own cost — with the designs that learn a lot but destroy more than they teach visibly dominated.
